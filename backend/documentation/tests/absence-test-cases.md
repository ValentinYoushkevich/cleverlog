# Тест-кейсы: Absence модуль

## GET /api/absences

**1. Пользователь видит только свои записи**
- Создать двух пользователей, каждому по несколько absences
- GET от первого пользователя
- Ожидать: 200, в ответе только записи первого пользователя

**2. Админ может получить записи конкретного пользователя через фильтр user_id**
- Создать пользователя и его absences
- GET от админа с `?user_id=<id>`
- Ожидать: 200, записи нужного пользователя

**3. Фильтрация по периоду date_from / date_to**
- Создать absences в разных месяцах
- GET с `?date_from=2025-01-01&date_to=2025-01-31`
- Ожидать: 200, только записи января

**4. Фильтрация по типу**
- Создать absences типов `vacation` и `sick`
- GET с `?type=sick`
- Ожидать: 200, только sick записи

**5. Ответ содержит duration_hours**
- Создать absence с `duration_days: 1`
- GET
- Ожидать: поле `duration_hours` присутствует и корректно конвертировано

**6. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

**7. Пагинация работает корректно**
- Создать 10 absences
- GET с `?page=1&limit=5`
- Ожидать: 200, `data.length === 5`, `pagination.total === 10`, `pagination.total_pages === 2`

---

## POST /api/absences

**8. Успешное создание на один рабочий день**
- POST с `date_from` и `date_to` на один рабочий день (понедельник)
- Ожидать: 200, `created.length === 1`, `skipped` все пустые

**9. Диапазон с выходными — выходные автоматически пропускаются**
- POST с диапазоном пн–вс (7 дней)
- Ожидать: 200, `created` содержит только 5 рабочих дней, `skipped.weekends` содержит 2 даты

**10. Дни с существующим work_log пропускаются**
- Создать work_log пользователя на среду
- POST с диапазоном пн–пт
- Ожидать: 200, среда в `skipped.work_logs`, остальные рабочие дни в `created`

**11. Дни с существующим absence пропускаются**
- Создать absence на вторник
- POST с диапазоном пн–пт
- Ожидать: 200, вторник в `skipped.absences`

**12. Все дни заняты — возвращает ошибку**
- Создать work_logs на все рабочие дни диапазона
- POST с этим диапазоном
- Ожидать: 400, код `ALL_DAYS_SKIPPED`, в ответе `skipped` со всеми датами

**13. Диапазон полностью из выходных**
- POST с `date_from` и `date_to` на субботу-воскресенье
- Ожидать: 400, код `NO_WORKING_DAYS`

**14. Закрытый месяц блокирует создание**
- Закрыть месяц через month_closures
- POST с датой в закрытом месяце
- Ожидать: 403 (или 400), ошибка закрытого месяца

**15. Без токена возвращает 401**
- POST без cookie
- Ожидать: 401

---

## PATCH /api/absences/:id

**16. Успешное изменение типа**
- Создать absence типа `vacation`
- PATCH с `{ type: 'sick' }`
- Ожидать: 200, `type === 'sick'`

**17. Успешный перенос даты на другой рабочий день**
- Создать absence на понедельник
- PATCH с `{ date: '<следующий вторник>' }`
- Ожидать: 200, дата обновлена

**18. Перенос на выходной запрещён**
- Создать absence
- PATCH с датой на субботу
- Ожидать: 400, код `WEEKEND_DATE`

**19. Перенос на день с work_log запрещён**
- Создать absence и work_log на другой рабочий день
- PATCH с датой дня с work_log
- Ожидать: 400, код `WORK_LOG_CONFLICT`

**20. Перенос на день с другим absence запрещён**
- Создать два absence
- PATCH первого на дату второго
- Ожидать: 400, код `ABSENCE_CONFLICT`

**21. Пользователь не может редактировать чужую запись**
- Создать absence от имени первого пользователя
- PATCH от второго пользователя (не admin)
- Ожидать: 403, код `FORBIDDEN`

**22. Админ может редактировать чужую запись**
- Создать absence от обычного пользователя
- PATCH от admin
- Ожидать: 200

**23. Несуществующий id возвращает 404**
- PATCH с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

**24. Закрытый месяц блокирует редактирование**
- Создать absence, закрыть месяц
- PATCH этой записи
- Ожидать: 403 (или 400), ошибка закрытого месяца

---

## DELETE /api/absences/:id

**25. Успешное удаление своей записи**
- Создать absence
- DELETE
- Ожидать: 200, `{ message: 'Запись удалена' }`
- Проверить: запись отсутствует в БД

**26. Пользователь не может удалить чужую запись**
- Создать absence от первого пользователя
- DELETE от второго пользователя (не admin)
- Ожидать: 403, код `FORBIDDEN`

**27. Админ может удалить чужую запись**
- Создать absence от обычного пользователя
- DELETE от admin
- Ожидать: 200

**28. Несуществующий id возвращает 404**
- DELETE с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

**29. Закрытый месяц блокирует удаление**
- Создать absence, закрыть месяц
- DELETE этой записи
- Ожидать: 403 (или 400), ошибка закрытого месяца
