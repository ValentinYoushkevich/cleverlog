# Тест-кейсы: MonthClosure модуль

## GET /api/month-closures/status/:year/:month
> Доступен всем авторизованным пользователям (только authenticate)

**1. Возвращает closed: true для закрытого месяца**
- Закрыть месяц через POST /month-closures
- GET /month-closures/status/2025/1
- Ожидать: 200, `{ closed: true }` (или аналогичная структура)

**2. Возвращает closed: false для открытого месяца**
- Не создавать запись закрытия
- GET /month-closures/status/2025/3
- Ожидать: 200, `{ closed: false }`

**3. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## GET /api/month-closures
> Только admin

**4. Возвращает список всех закрытых месяцев**
- Закрыть несколько месяцев
- GET /month-closures
- Ожидать: 200, массив с записями закрытий

**5. Возвращает пустой массив если ни одного закрытия нет**
- GET без предварительного закрытия
- Ожидать: 200, `[]`

**6. Обычный пользователь получает 403**
- GET от имени user
- Ожидать: 403

**7. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## POST /api/month-closures

**8. Успешное закрытие месяца**
- POST с `{ year: 2025, month: 1 }`
- Ожидать: 200, запись содержит `year`, `month`, `closed_by`
- Проверить: GET /status/2025/1 возвращает `closed: true`

**9. Повторное закрытие уже закрытого месяца возвращает 409**
- POST дважды с одинаковым year/month
- Ожидать: при втором запросе 409, код `ALREADY_CLOSED`

**10. После закрытия месяца создание work_log в нём блокируется**
- Закрыть месяц, попытаться создать work_log с датой в этом месяце
- Ожидать: ошибка от checkMonthClosed middleware

**11. После закрытия месяца создание absence в нём блокируется**
- Закрыть месяц, попытаться создать absence с date_from в этом месяце
- Ожидать: ошибка от checkMonthClosed middleware

**12. Обычный пользователь получает 403**
- POST от имени user
- Ожидать: 403

**13. Без токена возвращает 401**
- POST без cookie
- Ожидать: 401

---

## DELETE /api/month-closures/:year/:month

**14. Успешное открытие закрытого месяца**
- Закрыть месяц, затем DELETE /month-closures/2025/1
- Ожидать: 200, `{ message: 'Месяц 1/2025 открыт' }`
- Проверить: GET /status/2025/1 возвращает `closed: false`

**15. Попытка открыть не закрытый месяц возвращает 404**
- DELETE для месяца без записи закрытия
- Ожидать: 404, код `NOT_CLOSED`

**16. После открытия месяца создание work_log в нём снова разрешено**
- Закрыть месяц, открыть, создать work_log с датой в этом месяце
- Ожидать: work_log создан успешно

**17. Обычный пользователь получает 403**
- DELETE от имени user
- Ожидать: 403

**18. Без токена возвращает 401**
- DELETE без cookie
- Ожидать: 401
