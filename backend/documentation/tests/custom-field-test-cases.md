# Тест-кейсы: CustomField модуль

> Все роуты закрыты `authenticate + authorize('admin')`.
> Проверки 401/403 приведены по одному разу на группу — подразумевается что они применяются ко всем роутам.

---

## GET /api/custom-fields

**1. Возвращает список активных полей (без удалённых)**
- Создать 2 активных поля и 1 soft-deleted
- GET /custom-fields
- Ожидать: 200, в ответе 2 поля, удалённое не включено

**2. С параметром includeDeleted возвращает все поля**
- GET /custom-fields?includeDeleted=true
- Ожидать: 200, все 3 поля включая удалённое

**3. Обычный пользователь получает 403**
- GET от имени user
- Ожидать: 403

**4. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## POST /api/custom-fields

**5. Успешное создание поля типа Text**
- POST с `{ name: 'Задача', type: 'text' }`
- Ожидать: 200, поле создано, `id` присутствует

**6. Успешное создание поля типа Dropdown с опциями**
- POST с `{ name: 'Приоритет', type: 'dropdown', options: ['Низкий', 'Средний', 'Высокий'] }`
- Ожидать: 200, поле создано
- Проверить: GET /:id/options возвращает 3 опции

**7. Создание поля типа не-dropdown без опций — опции игнорируются**
- POST с `{ name: 'Флаг', type: 'checkbox', options: ['A', 'B'] }`
- Ожидать: 200, поле создано
- Проверить: GET /:id/options возвращает 400 (NOT_DROPDOWN)

**8. Невалидные данные не проходят Zod**
- POST без `name` или с неизвестным `type`
- Ожидать: 400 или 422

---

## PATCH /api/custom-fields/:id

**9. Успешное переименование поля**
- Создать поле, PATCH с `{ name: 'Новое имя' }`
- Ожидать: 200, `name` обновлён

**10. Смена типа разрешена если поле не используется**
- Создать поле типа `text`, не привязывать ни к одному work_log
- PATCH с `{ type: 'number' }`
- Ожидать: 200

**11. Смена типа запрещена если поле уже используется**
- Создать поле, привязать к work_log_custom_values
- PATCH с `{ type: 'number' }`
- Ожидать: 409, код `TYPE_CHANGE_FORBIDDEN`

**12. PATCH удалённого поля возвращает 400**
- Soft-delete поле, затем PATCH
- Ожидать: 400, код `FIELD_DELETED`

**13. Несуществующий id возвращает 404**
- PATCH с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

---

## DELETE /api/custom-fields/:id

**14. Успешный soft delete**
- Создать поле, DELETE
- Ожидать: 200, `{ message: 'Поле скрыто (soft delete)' }`
- Проверить: поле не возвращается в GET без includeDeleted

**15. Несуществующий id возвращает 404**
- DELETE с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

---

## POST /api/custom-fields/:id/restore

**16. Успешное восстановление удалённого поля**
- Soft-delete поле, затем POST /:id/restore
- Ожидать: 200, `{ message: 'Поле восстановлено' }`
- Проверить: поле снова появляется в GET без includeDeleted

**17. Restore не удалённого поля возвращает 400**
- POST /:id/restore для активного поля
- Ожидать: 400, код `NOT_DELETED`

**18. Несуществующий id возвращает 404**
- POST /:id/restore с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

---

## GET /api/custom-fields/:id/options

**19. Успешно возвращает опции dropdown поля**
- Создать dropdown поле с опциями
- GET /:id/options
- Ожидать: 200, массив опций с `id`, `label`

**20. Для не-dropdown поля возвращает 400**
- Создать поле типа `text`
- GET /:id/options
- Ожидать: 400, код `NOT_DROPDOWN`

**21. Несуществующий id возвращает 404**
- GET с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

---

## POST /api/custom-fields/:id/options

**22. Успешное добавление опции к dropdown**
- Создать dropdown поле
- POST /:id/options с `{ label: 'Новая опция' }`
- Ожидать: 200, опция создана с корректным `sort_order`

**23. Добавление опции к не-dropdown возвращает 400**
- Создать поле типа `text`
- POST /:id/options
- Ожидать: 400, код `NOT_DROPDOWN`

---

## DELETE /api/custom-fields/:id/options/:optionId

**24. Успешное устаревание опции (deprecate)**
- Создать dropdown поле с опцией
- DELETE /:id/options/:optionId
- Ожидать: 200, `{ message: 'Опция помечена как устаревшая' }`
- Проверить: опция помечена `deprecated`, но не удалена из БД

**25. Несуществующая опция возвращает 404**
- DELETE с несуществующим optionId
- Ожидать: 404, код `NOT_FOUND`

**26. Опция принадлежащая другому полю возвращает 404**
- Создать два dropdown поля с опциями
- DELETE /:id1/options/:optionId2 (опция от второго поля)
- Ожидать: 404, код `NOT_FOUND`
