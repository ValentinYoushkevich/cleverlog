# Тест-кейсы: Calendar модуль

## GET /api/calendar/:year/:month

**1. Успешно возвращает данные месяца**
- GET /calendar/2025/1
- Ожидать: 200, тело содержит `year`, `month`, `norm_hours`, `days` (массив)

**2. Возвращает дефолтную норму 168 если норма не задана**
- Не создавать запись нормы для месяца
- GET /calendar/2025/3
- Ожидать: 200, `norm_hours === 168`

**3. Возвращает кастомную норму если она задана**
- Создать норму для месяца через CalendarRepository (или PUT /norm)
- GET /calendar/2025/3
- Ожидать: 200, `norm_hours` равен установленному значению

**4. Дни содержат корректные статусы с учётом overrides**
- Создать override для конкретной даты (например, рабочий день → выходной)
- GET /calendar/2025/1
- Ожидать: 200, день с этой датой имеет изменённый `day_type`

**5. Некорректный месяц возвращает 400**
- GET /calendar/2025/13
- Ожидать: 400, код `INVALID_PERIOD`

**6. Некорректный год возвращает 400**
- GET /calendar/abc/1
- Ожидать: 400

**7. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## GET /api/calendar/norm/:year/:month

**8. Возвращает норму для месяца**
- Создать норму для 2025/6
- GET /calendar/norm/2025/6
- Ожидать: 200, `{ year: 2025, month: 6, norm_hours: <значение> }`

**9. Возвращает дефолт 168 если норма не задана**
- GET /calendar/norm/2025/9 (без записи в БД)
- Ожидать: 200, `norm_hours === 168`

**10. Некорректный месяц возвращает 400**
- GET /calendar/norm/2025/0
- Ожидать: 400, код `INVALID_PERIOD`

**11. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## PATCH /api/calendar/days/:date

**12. Админ успешно меняет статус дня**
- PATCH /calendar/days/2025-01-06 с `{ day_type: 'holiday' }`
- Ожидать: 200, ответ содержит обновлённый день

**13. Повторный PATCH обновляет существующий override (upsert)**
- PATCH дату дважды с разными day_type
- Ожидать: 200 оба раза, финальный статус соответствует второму запросу

**14. Обычный пользователь получает 403**
- PATCH от имени user (не admin)
- Ожидать: 403

**15. Без токена возвращает 401**
- PATCH без cookie
- Ожидать: 401

**16. Невалидный day_type не проходит валидацию**
- PATCH с `{ day_type: 'unknown_type' }`
- Ожидать: 400 или 422

---

## PUT /api/calendar/norm/:year/:month

**17. Админ успешно устанавливает норму**
- PUT /calendar/norm/2025/4 с `{ norm_hours: 160 }`
- Ожидать: 200, `norm_hours === 160`

**18. Повторный PUT обновляет существующую норму (upsert)**
- PUT дважды с разными значениями
- Ожидать: 200 оба раза, финальное значение соответствует второму запросу

**19. Обычный пользователь получает 403**
- PUT от имени user
- Ожидать: 403

**20. Без токена возвращает 401**
- PUT без cookie
- Ожидать: 401

**21. Некорректный месяц возвращает 400**
- PUT /calendar/norm/2025/13 с `{ norm_hours: 160 }`
- Ожидать: 400, код `INVALID_PERIOD`
