# Тест-кейсы: Report модуль

> `GET /user` и `GET /user/export` — доступны всем авторизованным.
> Остальные роуты — только admin.

---

## GET /api/reports/user

**1. Пользователь получает свой отчёт**
- Создать логи и absences для пользователя
- GET /reports/user?date_from=2025-01-01&date_to=2025-01-31
- Ожидать: 200, `rows` содержат записи, `totals` заполнены

**2. Пользователь не видит чужие данные**
- Создать логи для двух пользователей
- GET от первого без фильтра user_id
- Ожидать: 200, только записи первого пользователя

**3. Админ может получить отчёт по конкретному пользователю**
- GET от admin с `?user_id=<id>`
- Ожидать: 200, только записи нужного пользователя

**4. Фильтрация по project_id**
- GET с `?project_id=<id>`
- Ожидать: 200, только логи нужного проекта в `rows`

**5. Фильтрация по type=work возвращает только work_logs**
- GET с `?type=work`
- Ожидать: 200, в `rows` нет записей типа vacation/sick_leave

**6. Фильтрация по type=vacation возвращает только absences типа vacation**
- GET с `?type=vacation`
- Ожидать: 200, в `rows` только vacation записи

**7. Фильтрация по task_number (подстрока)**
- Создать лог с `task_number: 'PROJ-42'`
- GET с `?task_number=PROJ`
- Ожидать: 200, лог найден

**8. norm_block присутствует при запросе за полный месяц**
- GET с `?date_from=2025-01-01&date_to=2025-01-31`
- Ожидать: `norm_block` содержит `norm`, `fact`, `deviation`

**9. norm_block отсутствует при запросе за неполный период**
- GET с `?date_from=2025-01-05&date_to=2025-01-20`
- Ожидать: `norm_block === null`

**10. totals.by_project содержит корректную разбивку часов**
- Создать логи на два проекта
- GET
- Ожидать: `totals.by_project` содержит оба проекта с правильными суммами

**11. Строки отсортированы по дате**
- Создать логи в разном порядке дат
- GET
- Ожидать: `rows` отсортированы по `date` возрастающе

**12. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

## GET /api/reports/user/export

**13. Успешно возвращает Excel-файл**
- Создать логи для пользователя
- GET /reports/user/export
- Ожидать: 200, `Content-Type` xlsx, тело непустое

**14. Фильтры применяются к экспорту**
- GET /reports/user/export?date_from=2025-01-01&date_to=2025-01-31
- Ожидать: 200, файл сформирован (размер > 0)

**15. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

## GET /api/reports/project
> Только admin

**16. Успешно возвращает отчёт по проекту**
- Создать логи нескольких пользователей на один проект
- GET /reports/project?project_id=<id>
- Ожидать: 200, `rows` с разбивкой по пользователям, `totals.by_user`

**17. Фильтрация по user_id внутри проекта**
- GET с `?project_id=<id>&user_id=<id>`
- Ожидать: 200, только логи нужного пользователя

**18. totals.by_user содержит корректные суммы**
- Создать логи двух пользователей на проект с известными часами
- GET
- Ожидать: `totals.by_user` содержит оба имени с правильными суммами

**19. Обычный пользователь получает 403**
- GET от имени user
- Ожидать: 403

## GET /api/reports/monthly-summary
> Только admin

**20. Успешно возвращает сводный отчёт за месяц**
- Создать пользователей, проекты, логи и absences за месяц
- GET /reports/monthly-summary?year=2025&month=1
- Ожидать: 200, `rows` по каждому пользователю, `projects`, `totals`, `norm`

**21. Каждая строка содержит by_project, absence_hours, fact_hours, is_on_norm**
- GET
- Ожидать: каждый элемент `rows` содержит все обязательные поля

**22. totals суммирует данные всех пользователей**
- Создать двух пользователей с логами
- GET
- Ожидать: `totals.fact_hours` равен сумме `fact_hours` всех строк

**23. Неактивные пользователи не попадают в отчёт**
- Деактивировать пользователя
- GET
- Ожидать: деактивированный пользователь отсутствует в `rows`

**24. Обычный пользователь получает 403**
- GET от имени user
- Ожидать: 403

## GET /api/reports/unlogged
> Только admin

**25. Возвращает только пользователей с незаполненными днями**
- Создать пользователя с незаполненными рабочими днями
- GET /reports/unlogged?year=2025&month=1
- Ожидать: 200, пользователь присутствует в `users`

**26. Пользователь с полностью заполненным месяцем не попадает в список**
- Создать пользователя, заполнить все рабочие дни
- GET
- Ожидать: пользователь отсутствует в `users`

**27. Каждая запись содержит unlogged_count, unlogged_dates, last_log_date**
- GET
- Ожидать: каждый элемент содержит все обязательные поля

**28. count соответствует количеству элементов в users**
- GET
- Ожидать: `count === users.length`

**29. Обычный пользователь получает 403**
- GET от имени user
- Ожидать: 403
