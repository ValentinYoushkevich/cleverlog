# Тест-кейсы: WorkLog модуль

## GET /api/work-logs
> Доступен всем авторизованным

**1. Пользователь видит только свои логи**
- Создать логи для двух пользователей
- GET от первого пользователя
- Ожидать: 200, только логи первого пользователя

**2. Админ может получить логи конкретного пользователя через фильтр user_id**
- GET от admin с `?user_id=<id>`
- Ожидать: 200, логи нужного пользователя

**3. Фильтрация по project_id**
- Создать логи на разные проекты
- GET с `?project_id=<id>`
- Ожидать: 200, только логи нужного проекта

**4. Фильтрация по периоду date_from / date_to**
- Создать логи в разные месяцы
- GET с `?date_from=2025-01-01&date_to=2025-01-31`
- Ожидать: 200, только январские логи

**5. Фильтрация по task_number (подстрока)**
- Создать лог с `task_number: 'PROJ-123'`
- GET с `?task_number=PROJ`
- Ожидать: 200, лог найден

**6. Ответ содержит duration_hours и custom_fields**
- Создать лог с кастомными полями
- GET
- Ожидать: каждый лог содержит `duration_hours`, `custom_fields`

**7. Пагинация работает корректно**
- Создать 10 логов
- GET с `?page=1&limit=5`
- Ожидать: 200, `data.length === 5`, `pagination.total === 10`

**8. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## POST /api/work-logs

**9. Успешное создание лога**
- POST с `{ project_id, date, duration: '8h', comment, task_number }`
- Ожидать: 200, лог создан с `duration_hours === 8`, `warning: null`

**10. Создание лога в будущем запрещено**
- POST с `date` завтрашнего дня
- Ожидать: 400, код `FUTURE_DATE`

**11. Проект не существует возвращает 404**
- POST с несуществующим `project_id`
- Ожидать: 404, код `PROJECT_NOT_FOUND`

**12. Неактивный проект блокирует создание**
- Создать проект со статусом `on_hold` или `closed`
- POST с этим `project_id`
- Ожидать: 400, код `PROJECT_NOT_ACTIVE`

**13. Наличие absence в этот день блокирует создание**
- Создать absence на дату
- POST work_log на ту же дату
- Ожидать: 400, код `ABSENCE_CONFLICT`

**14. Превышение 12 часов в день возвращает warning**
- Создать лог на 8h, создать ещё один лог на 5h на тот же день
- Ожидать: 200, второй лог создан, `warning` содержит сообщение о превышении

**15. Создание с кастомными полями**
- Привязать к проекту required кастомное поле
- POST с заполненным `custom_fields`
- Ожидать: 200, лог создан

**16. Незаполненное required кастомное поле блокирует создание**
- Привязать к проекту required кастомное поле
- POST без `custom_fields` или с пустым значением
- Ожидать: 400, код `REQUIRED_FIELD_MISSING`

**17. Закрытый месяц блокирует создание**
- Закрыть месяц, POST с датой в нём
- Ожидать: 403 (или 400), ошибка закрытого месяца

**18. Админ может создать лог от имени другого пользователя**
- POST от admin с `target_user_id`
- Ожидать: 200, лог создан с `user_id === target_user_id`

**19. Без токена возвращает 401**
- POST без cookie
- Ожидать: 401

## PATCH /api/work-logs/:id

**20. Успешное обновление комментария**
- PATCH с `{ comment: 'Новый комментарий' }`
- Ожидать: 200, `comment` обновлён

**21. Успешное изменение даты**
- PATCH с `{ date: '<другой рабочий день>' }`
- Ожидать: 200, дата обновлена

**22. Смена даты на будущую запрещена**
- PATCH с `{ date: '<завтра>' }`
- Ожидать: 400, код `FUTURE_DATE`

**23. Смена проекта на неактивный запрещена**
- PATCH с `{ project_id: <closed_project_id> }`
- Ожидать: 400, код `PROJECT_NOT_ACTIVE`

**24. Смена даты на день с absence запрещена**
- Создать absence на другую дату, PATCH лога с этой датой
- Ожидать: 400, код `ABSENCE_CONFLICT`

**25. Пользователь не может редактировать чужой лог**
- PATCH от второго пользователя на лог первого
- Ожидать: 403, код `FORBIDDEN`

**26. Админ может редактировать чужой лог**
- PATCH от admin на лог обычного пользователя
- Ожидать: 200

**27. Несуществующий id возвращает 404**
- PATCH с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

**28. Закрытый месяц блокирует редактирование**
- Создать лог, закрыть месяц, PATCH
- Ожидать: ошибка закрытого месяца

---

## DELETE /api/work-logs/:id

**29. Успешное удаление своего лога**
- DELETE
- Ожидать: 200, `{ message: 'Лог удалён' }`
- Проверить: GET больше не возвращает этот лог

**30. Пользователь не может удалить чужой лог**
- DELETE от второго пользователя на лог первого
- Ожидать: 403, код `FORBIDDEN`

**31. Админ может удалить чужой лог**
- DELETE от admin на лог обычного пользователя
- Ожидать: 200

**32. Несуществующий id возвращает 404**
- DELETE с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

**33. Закрытый месяц блокирует удаление**
- Создать лог, закрыть месяц, DELETE
- Ожидать: ошибка закрытого месяца

**34. Без токена возвращает 401**
- DELETE без cookie
- Ожидать: 401
