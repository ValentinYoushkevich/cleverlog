# Тест-кейсы: ProjectCustomField модуль

> Роуты монтируются как вложенные: `/api/projects/:projectId/custom-fields`.
> GET доступен всем авторизованным, остальные — только admin.

---

## GET /api/projects/:projectId/custom-fields

**1. Возвращает список полей привязанных к проекту**
- Создать проект и привязать к нему 2 кастомных поля
- GET /projects/:projectId/custom-fields
- Ожидать: 200, массив из 2 полей с `is_required`, `is_enabled`

**2. Возвращает пустой массив если полей нет**
- Создать проект без привязанных полей
- GET
- Ожидать: 200, `[]`

**3. Не возвращает поля других проектов**
- Создать два проекта, привязать поля к каждому
- GET для первого проекта
- Ожидать: только поля первого проекта

**4. Обычный пользователь может получить список (только authenticate)**
- GET от имени user
- Ожидать: 200

**5. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## POST /api/projects/:projectId/custom-fields

**6. Успешная привязка поля к проекту**
- Создать проект и кастомное поле
- POST с `{ custom_field_id, is_required: false, is_enabled: true }`
- Ожидать: 200, привязка создана
- Проверить: GET возвращает это поле

**7. Повторная привязка одного и того же поля возвращает 409**
- Привязать поле, затем POST снова с тем же field_id
- Ожидать: 409, код `ALREADY_ATTACHED`

**8. Привязка soft-deleted поля возвращает 404**
- Soft-delete кастомное поле
- POST с его id
- Ожидать: 404, код `NOT_FOUND`

**9. Привязка несуществующего поля возвращает 404**
- POST с несуществующим custom_field_id
- Ожидать: 404, код `NOT_FOUND`

**10. Обычный пользователь получает 403**
- POST от имени user
- Ожидать: 403

**11. Без токена возвращает 401**
- POST без cookie
- Ожидать: 401

---

## PATCH /api/projects/:projectId/custom-fields/:fieldId

**12. Успешное обновление is_required у привязанного поля**
- Привязать поле с `is_required: false`
- PATCH с `{ is_required: true }`
- Ожидать: 200, `is_required === true`

**13. Успешное обновление is_enabled у привязанного поля**
- PATCH с `{ is_enabled: false }`
- Ожидать: 200, `is_enabled === false`

**14. PATCH для не привязанного поля создаёт привязку (upsert)**
- Создать поле без привязки к проекту
- PATCH /projects/:projectId/custom-fields/:fieldId с `{ is_enabled: true }`
- Ожидать: 200, привязка создана с дефолтными значениями + переданными данными

**15. PATCH для soft-deleted поля без привязки возвращает 404**
- Soft-delete поле, попытаться PATCH без существующей привязки
- Ожидать: 404, код `NOT_FOUND`

**16. Обычный пользователь получает 403**
- PATCH от имени user
- Ожидать: 403

**17. Без токена возвращает 401**
- PATCH без cookie
- Ожидать: 401

---

## DELETE /api/projects/:projectId/custom-fields/:fieldId

**18. Успешное открепление поля от проекта**
- Привязать поле к проекту
- DELETE /projects/:projectId/custom-fields/:fieldId
- Ожидать: 200, `{ message: 'Поле откреплено от проекта' }`
- Проверить: GET больше не возвращает это поле

**19. Открепление не привязанного поля возвращает 404**
- DELETE для поля которое не привязано к проекту
- Ожидать: 404, код `NOT_FOUND`

**20. Открепление поля не затрагивает его привязку к другим проектам**
- Привязать одно поле к двум проектам
- DELETE для первого проекта
- Проверить: GET для второго проекта всё ещё возвращает это поле

**21. Обычный пользователь получает 403**
- DELETE от имени user
- Ожидать: 403

**22. Без токена возвращает 401**
- DELETE без cookie
- Ожидать: 401
