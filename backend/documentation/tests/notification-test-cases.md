# Тест-кейсы: Notification модуль

> Все роуты закрыты `authenticate + authorize('admin')`.

---

## GET /api/notifications/settings

**1. Возвращает глобальные настройки уведомлений**
- GET /notifications/settings
- Ожидать: 200, `{ global_enabled: true }` (дефолт если запись не создана)

**2. Возвращает актуальное значение после обновления**
- PATCH /notifications/settings с `{ enabled: false }`
- GET /notifications/settings
- Ожидать: 200, `{ global_enabled: false }`

**3. Обычный пользователь получает 403**
- GET от имени user
- Ожидать: 403

**4. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## PATCH /api/notifications/settings

**5. Успешное отключение глобальных уведомлений**
- PATCH с `{ enabled: false }`
- Ожидать: 200, `global_enabled === false`

**6. Успешное включение глобальных уведомлений**
- PATCH с `{ enabled: false }`, затем PATCH с `{ enabled: true }`
- Ожидать: 200, `global_enabled === true`

**7. Повторный PATCH работает как upsert**
- PATCH дважды с разными значениями
- Ожидать: 200 оба раза, финальное значение соответствует второму запросу

**8. Обычный пользователь получает 403**
- PATCH от имени user
- Ожидать: 403

**9. Без токена возвращает 401**
- PATCH без cookie
- Ожидать: 401

---

## PATCH /api/notifications/users/:userId

**10. Успешное отключение уведомлений для конкретного пользователя**
- Создать пользователя
- PATCH /notifications/users/:userId с `{ enabled: false }`
- Ожидать: 200, `enabled === false`

**11. Успешное включение уведомлений для пользователя**
- PATCH с `{ enabled: false }`, затем PATCH с `{ enabled: true }`
- Ожидать: 200, `enabled === true`

**12. Повторный PATCH работает как upsert**
- PATCH дважды с разными значениями
- Ожидать: 200 оба раза, финальное значение соответствует второму запросу

**13. Обычный пользователь получает 403**
- PATCH от имени user
- Ожидать: 403

**14. Без токена возвращает 401**
- PATCH без cookie
- Ожидать: 401

## POST /api/notifications/trigger

> Ручной запуск рассылки. Так как реальная отправка email нежелательна в тестах,
> проверяем только поведение эндпоинта, не содержимое писем.
> Для изоляции от реального mailer — мокировать `mailer.sendMail` через vi.mock или передавать stub.

**15. Успешный вызов trigger возвращает 200**
- POST /notifications/trigger
- Ожидать: 200 (независимо от того отправились ли письма)

**16. Обычный пользователь получает 403**
- POST от имени user
- Ожидать: 403

**17. Без токена возвращает 401**
- POST без cookie
- Ожидать: 401
