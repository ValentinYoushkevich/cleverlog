# Тест-кейсы: Auth модуль

## POST /api/auth/register

**1. Успешная регистрация по валидному инвайту**
- Создать пользователя с `invite_token_hash` и `invite_expires_at` в будущем, email не задан админом
- POST с валидным токеном, email и сильным паролем
- Ожидать: 200, `{ message: 'Регистрация успешна' }`
- Проверить: `invite_token_hash` в БД стал `null`, `password_hash` установлен

**2. Невалидный токен инвайта**
- POST с несуществующим токеном
- Ожидать: 400, код `INVALID_TOKEN`

**3. Истёкший токен инвайта**
- Создать пользователя с `invite_expires_at` в прошлом
- POST с этим токеном
- Ожидать: 400, код `TOKEN_EXPIRED`

**4. Email не совпадает с тем что задал админ**
- Создать пользователя с заранее заданным `email` администратором
- POST с тем же токеном, но другим email в теле
- Ожидать: 400, код `EMAIL_MISMATCH`

**5. Email уже занят другим пользователем**
- Создать двух пользователей: один уже зарегистрирован с email, второй — с инвайтом без email
- POST для второго с email первого
- Ожидать: 409, код `EMAIL_EXISTS`

**6. Слабый пароль не проходит валидацию Zod**
- POST с валидным токеном и паролем типа `123`
- Ожидать: 422 (или 400), ошибка валидации пароля

## POST /api/auth/login

**7. Успешный логин**
- Создать активного пользователя с паролем
- POST с верными email и password
- Ожидать: 200, HttpOnly cookie установлена, тело содержит `user.id`, `user.email`, `user.role`

**8. Неверный пароль**
- POST с верным email, неверным паролем
- Ожидать: 401, код `INVALID_CREDENTIALS`
- Проверить: `failed_attempts` в БД увеличился на 1

**9. Пользователь не существует**
- POST с несуществующим email
- Ожидать: 401, код `INVALID_CREDENTIALS`

**10. Аккаунт неактивен**
- Создать пользователя со статусом `inactive`
- POST с верными данными
- Ожидать: 403, код `ACCOUNT_INACTIVE`

**11. Аккаунт заблокирован после 5 неверных попыток**
- Создать пользователя, выполнить 5 POST с неверным паролем
- 6-й POST с любым паролем
- Ожидать: 429, код `ACCOUNT_LOCKED`

**12. Успешный логин сбрасывает счётчик failed_attempts**
- Создать пользователя с `failed_attempts: 3`
- POST с верным паролем
- Ожидать: 200
- Проверить: `failed_attempts` в БД стал `0`

## POST /api/auth/logout

**13. Успешный логаут очищает cookie**
- Залогиниться, получить cookie
- POST /logout с cookie
- Ожидать: 200, Set-Cookie очищает токен (пустое значение или expires в прошлом)

**14. Логаут без токена возвращает 200**
- POST /logout без cookie и без заголовка Authorization
- Ожидать: 200

---

## GET /api/auth/me

**15. Возвращает данные текущего пользователя**
- Залогиниться
- GET /me с cookie
- Ожидать: 200, тело содержит `id`, `email`, `role`, `first_name`, `last_name`

**16. Без токена возвращает 401**
- GET /me без cookie
- Ожидать: 401

---

## PATCH /api/auth/profile

**17. Успешное обновление профиля**
- Залогиниться
- PATCH с `{ first_name, last_name, position }`
- Ожидать: 200, тело содержит обновлённые поля

**18. Без токена возвращает 401**
- PATCH без cookie
- Ожидать: 401

**19. Невалидные данные не проходят Zod**
- Залогиниться
- PATCH с `{ first_name: '' }` или слишком длинной строкой
- Ожидать: 422 (или 400), ошибка валидации

**20. position можно сбросить в null**
- Залогиниться, установить position
- PATCH с `{ position: null }`
- Ожидать: 200, `position: null` в ответе

---

## POST /api/auth/change-password

**21. Успешная смена пароля**
- Залогиниться
- POST с верным `current_password` и сильным `new_password`
- Ожидать: 200, `{ message: 'Пароль изменён' }`
- Проверить: старый пароль больше не работает для логина

**22. Неверный текущий пароль**
- Залогиниться
- POST с неверным `current_password`
- Ожидать: 400, код `INVALID_PASSWORD`

**23. Новый пароль не проходит политику Zod**
- Залогиниться
- POST с верным `current_password` и слабым `new_password`
- Ожидать: 422 (или 400), ошибка валидации

**24. Без токена возвращает 401**
- POST без cookie
- Ожидать: 401
