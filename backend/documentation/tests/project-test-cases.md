# Тест-кейсы: Project модуль

## GET /api/projects
> Доступен всем авторизованным пользователям

**1. Успешно возвращает список проектов**
- Создать несколько проектов
- GET /projects
- Ожидать: 200, массив проектов с `id`, `name`, `status`

**2. Фильтрация по статусу active**
- Создать проекты со статусами `active`, `on_hold`, `closed`
- GET /projects?status=active
- Ожидать: 200, только активные проекты

**3. Фильтрация по статусу on_hold**
- GET /projects?status=on_hold
- Ожидать: 200, только проекты со статусом on_hold

**4. Без фильтра возвращает все проекты**
- GET /projects без параметра status
- Ожидать: 200, все проекты независимо от статуса

**5. Обычный пользователь видит список проектов**
- GET от имени user
- Ожидать: 200

**6. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## GET /api/projects/:id

**7. Успешно возвращает проект по id**
- Создать проект
- GET /projects/:id
- Ожидать: 200, объект с `id`, `name`, `status`

**8. Несуществующий id возвращает 404**
- GET с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

**9. Обычный пользователь может получить проект**
- GET от имени user
- Ожидать: 200

**10. Без токена возвращает 401**
- GET без cookie
- Ожидать: 401

---

## POST /api/projects
> Только admin

**11. Успешное создание проекта**
- POST с `{ name: 'Новый проект' }`
- Ожидать: 200, проект создан с `id`, `name`, дефолтный `status: 'active'`

**12. Невалидные данные не проходят Zod**
- POST без `name` или с пустой строкой
- Ожидать: 400 или 422

**13. Обычный пользователь получает 403**
- POST от имени user
- Ожидать: 403

**14. Без токена возвращает 401**
- POST без cookie
- Ожидать: 401

---

## PATCH /api/projects/:id
> Только admin

**15. Успешное переименование проекта**
- Создать проект, PATCH с `{ name: 'Новое название' }`
- Ожидать: 200, `name` обновлён

**16. Смена статуса с active на on_hold**
- PATCH с `{ status: 'on_hold' }`
- Ожидать: 200, `status === 'on_hold'`

**17. Смена статуса с on_hold обратно на active**
- Создать проект со статусом `on_hold`, PATCH с `{ status: 'active' }`
- Ожидать: 200, `status === 'active'`

**18. Смена статуса на closed**
- PATCH с `{ status: 'closed' }`
- Ожидать: 200, `status === 'closed'`

**19. Закрытый проект нельзя использовать для нового work_log**
- Закрыть проект, попытаться создать work_log с этим project_id
- Ожидать: 400, код `PROJECT_NOT_ACTIVE`

**20. Несуществующий id возвращает 404**
- PATCH с несуществующим id
- Ожидать: 404, код `NOT_FOUND`

**21. Обычный пользователь получает 403**
- PATCH от имени user
- Ожидать: 403

**22. Без токена возвращает 401**
- PATCH без cookie
- Ожидать: 401
